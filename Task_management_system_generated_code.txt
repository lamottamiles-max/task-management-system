# Task Management System

## Overview
This project is a simple task tracker web application designed for teams to manage tasks effectively. Users can create, assign, and mark tasks as complete while viewing the status of tasks through a dashboard.

## Project Structure
Below is the file structure for the project:
```
task-management-system/
├── client/                      # React frontend
│   ├── public/                  # Static files
│   ├── src/
│   │   ├── components/          # React components
│   │   ├── pages/               # React pages
│   │   ├── App.js               # Main app component
│   │   ├── index.js             # Entry point
│   ├── package.json             # Client dependencies
├── server/                      # Node.js backend
│   ├── controllers/             # Request handlers

│   ├── package.json             # Server dependencies
├── .env                         # Environment variables
├── README.md                    # Project documentation
```

## Technologies Used
- **Frontend**: React.js
- **Backend**: Node.js with Express
- **Database**: MongoDB (with Mongoose)
- **Deployment**: Vercel / AWS

## Step-by-Step Setup Instructions

### Prerequisites
- Node.js (>= 14.x)
- npm or yarn
- MongoDB instance (e.g., MongoDB Atlas)

### Install Dependencies
1. Clone the repository:
   ```bash
   git clone https://github.com/yourusername/task-management-system.git
   cd task-management-system
   ```

2. Navigate to the server folder and install dependencies:
   ```bash
   cd server
   npm install
   ```

3. Navigate to the client folder and install dependencies:
   ```bash
   cd ../client
   npm install
   ```

### Environment Variables
Create a `.env` file in the `server` directory with the following contents:
```bash
PORT=5000                      # Port the server will run on
MONGO_URI=mongodb://<username>:<password>@<host>:<port>/<dbname>  # MongoDB URI
```

### Running the Application
1. Start the server:
   ```bash
   cd server
   npm start
   ```

2. Start the client:
   ```bash
   cd ../client
   npm start
   ```

Navigate to `http://localhost:3000` in your web browser to view the application.

## Deployment Guide

### Deploying Backend (Node.js)
1. Choose a hosting service like AWS or Heroku.
2. Configure environment variables in your hosting dashboard.
3. Push your Node.js codebase to the remote repository linked with your host, then follow their deployment guide.

### Deploying Frontend (React)
1. Choose a service like Vercel or Netlify.
2. Connect your GitHub repository.
3. Set up build settings with `npm run build` for production.
4. Specify the output directory as `client/build`.

## API Documentation

### API Endpoints
- **GET /tasks**: Get a list of all tasks.
- **POST /tasks**: Create a new task.
- **PATCH /tasks/:id**: Update task status (complete/incomplete).
- **DELETE /tasks/:id**: Delete a task.

**Example of a Task Object**:
```json
{
  "_id": "task_id",
  "title": "Task Title",
  "assignedTo": "user_id",
  "dueDate": "2023-12-31T23:59:59.999Z",
  "completed": false
}
```

## Architecture Explanation
This application employs a client-server architecture:
- **Frontend**: React.js for building a dynamic user interface that communicates with the backend API.
- **Backend**: Node.js with Express handles HTTP requests, processes data, and interacts with MongoDB.
- **Database**: MongoDB stores task data, facilitating easy retrieval and updates.
- **Deployment**: Both the frontend and backend are independently deployable, ensuring scalability.

## Example Code

### Server: Express Application (`server/app.js`)
```javascript
const express = require('express');
const mongoose = require('mongoose');
const taskRoutes = require('./routes/taskRoutes');
const app = express();
require('dotenv').config();

app.use(express.json());
app.use('/api/tasks', taskRoutes);

// MongoDB Connection
mongoose.connect(process.env.MONGO_URI, { useNewUrlParser: true, useUnifiedTopology: true })
    .then(() => console.log('MongoDB connected'))
    .catch(err => console.error(err));

// Error Handling Middleware
app.use((err, req, res, next) => {
    console.error(err.message);
    res.status(500).send('Server Error');
});

// Start Server
const PORT = process.env.PORT || 5000;
app.listen(PORT, () => {
    console.log(`Server running on port ${PORT}`);
});
```

### Frontend: Task Component (`client/src/components/Task.js`)
```javascript
import React from 'react';

const Task = ({ task, onComplete }) => {
    return (
        <div className={`task ${task.completed ? 'completed' : ''}`}>
            <h3>{task.title}</h3>
            <p>Assigned to: {task.assignedTo}</p>
            <p>Due Date: {new Date(task.dueDate).toLocaleDateString()}</p>
            <button onClick={() => onComplete(task._id)}>
                {task.completed ? 'Undo' : 'Complete'}
            </button>
        </div>
    );
};

export default Task;
```
